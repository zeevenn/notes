CSS 中的单位分为绝对单位和相对单位。相对单位是响应式设计的基础。

## 绝对单位

- **`px`**：像素，屏幕上最小的显示单位。在高 DPI（Retina）屏上，1 CSS px 对应多个物理像素，但视觉大小保持一致。

## 相对单位

### 字体相对

- **`em`**：相对于**当前元素**的 `font-size`。嵌套使用时会叠乘，容易引发"em 嵌套陷阱"。
- **`rem`**（root em）：相对于 **`html` 根元素**的 `font-size`（浏览器默认 16px）。适合整站字号和间距的统一缩放，是目前最推荐的相对字体单位。
- **`ch`**：当前字体中 `0` 字符的宽度，常用于设置输入框宽度（如 `width: 20ch` 刚好容纳 20 个等宽字符）。

### 视口相对

- **`vw`**：视口宽度的 1%，`100vw` = 视口全宽。
- **`vh`**：视口高度的 1%，`100vh` = 视口全高。
- **`vmin`**：`vw` 和 `vh` 中较小的一个。
- **`vmax`**：`vw` 和 `vh` 中较大的一个。

> [!TIP]
>
> **移动端陷阱**：`100vh` 在移动浏览器上包含地址栏高度，导致内容被遮挡。现代浏览器引入了新单位：
>
> - `svh`（small viewport height）：地址栏**展开**时的视口高度
> - `lvh`（large viewport height）：地址栏**收起**时的视口高度
> - `dvh`（dynamic viewport height）：随地址栏动态变化，移动端全屏场景推荐使用

### 百分比

- **`%`**：相对于**父元素**的对应属性值。`width: 50%` 是父元素宽度的一半；`font-size: 120%` 是父元素字号的 1.2 倍。

## 流式字号：clamp()

`clamp(min, preferred, max)` 实现在最小值和最大值之间随视口流式缩放，无需媒体查询：

```css
/* 字号在 16px 到 24px 之间，随视口宽度线性变化 */
font-size: clamp(1rem, 2.5vw, 1.5rem);

/* 容器宽度在 320px 到 1200px 之间响应式变化 */
width: clamp(320px, 80%, 1200px);
```

## 选择建议

| 场景                       | 推荐单位            |
| -------------------------- | ------------------- |
| 全站字号缩放               | `rem`               |
| 组件内相对字号             | `em`                |
| 视口全屏布局               | `vw` / `dvh`        |
| 流式字号/间距              | `clamp()`           |
| 固定像素细节（边框、阴影） | `px`                |
| 布局宽度                   | `%` 或 `fr`（Grid） |
